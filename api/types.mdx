## Client Configuration

### CalDAVOptions

Configuration options for the CalDAV client.

<ParamField path="baseUrl" type="string" required>
  The base URL of the CalDAV server (e.g., `https://caldav.example.com`)
</ParamField>

<ParamField path="auth" type="AuthOptions" required>
  Authentication credentials for the CalDAV server
</ParamField>

<ParamField path="requestTimeout" type="number">
  Timeout for HTTP requests in milliseconds
</ParamField>

<ParamField path="logRequests" type="boolean">
  Enable logging of HTTP requests for debugging
</ParamField>

<ParamField path="prodId" type="string">
  Product identifier for generated iCalendar data (PRODID property)
</ParamField>

<ParamField path="headers" type="AxiosHeaders">
  Additional HTTP headers to include in all requests
</ParamField>

## Calendar Types

### Calendar

Represents a CalDAV calendar resource.

<ResponseField name="displayName" type="string" required>
  Human-readable name of the calendar
</ResponseField>

<ResponseField name="url" type="string" required>
  URL of the calendar resource
</ResponseField>

<ResponseField name="ctag" type="string">
  Calendar collection tag for synchronization
</ResponseField>

<ResponseField name="supportedComponents" type="SupportedComponent[]" required>
  Array of component types supported by this calendar
</ResponseField>

<ResponseField name="color" type="string">
  Color code for the calendar (e.g., `#FF5733`)
</ResponseField>

### SupportedComponent

Component types that can be stored in a CalDAV calendar.

```typescript
type SupportedComponent = 
  | "VEVENT"      // Calendar events
  | "VTODO"       // Todo items
  | "VJOURNAL"    // Journal entries
  | "VFREEBUSY"   // Free/busy time
  | "VTIMEZONE";  // Timezone definitions
```

## Event Types

### Event

Represents a calendar event.

<ResponseField name="uid" type="string" required>
  Unique identifier for the event
</ResponseField>

<ResponseField name="summary" type="string" required>
  Title or summary of the event
</ResponseField>

<ResponseField name="start" type="Date" required>
  Start date and time of the event
</ResponseField>

<ResponseField name="end" type="Date" required>
  End date and time of the event
</ResponseField>

<ResponseField name="description" type="string">
  Detailed description of the event
</ResponseField>

<ResponseField name="location" type="string">
  Location where the event takes place
</ResponseField>

<ResponseField name="status" type="EventStatus">
  Status of the event (TENTATIVE, CONFIRMED, or CANCELLED)
</ResponseField>

<ResponseField name="etag" type="string" required>
  Entity tag for concurrency control
</ResponseField>

<ResponseField name="href" type="string" required>
  URL reference to the event resource
</ResponseField>

<ResponseField name="wholeDay" type="boolean">
  Whether this is an all-day event
</ResponseField>

<ResponseField name="recurrenceRule" type="RecurrenceRule">
  Recurrence rule for repeating events
</ResponseField>

<ResponseField name="startTzid" type="string">
  Timezone identifier for the start time (e.g., `America/New_York`)
</ResponseField>

<ResponseField name="endTzid" type="string">
  Timezone identifier for the end time (e.g., `America/New_York`)
</ResponseField>

<ResponseField name="alarms" type="Alarm[]">
  Array of alarm/reminder definitions
</ResponseField>

### EventStatus

Status values for calendar events.

```typescript
type EventStatus = "TENTATIVE" | "CONFIRMED" | "CANCELLED";
```

### EventRef

Reference to an event resource with metadata.

<ResponseField name="href" type="string" required>
  URL reference to the event resource
</ResponseField>

<ResponseField name="etag" type="string" required>
  Entity tag for version tracking
</ResponseField>

## Recurring Events

### RecurrenceRule

Defines recurrence patterns for repeating events.

<ParamField path="freq" type="'DAILY' | 'WEEKLY' | 'MONTHLY' | 'YEARLY'">
  Frequency of recurrence
</ParamField>

<ParamField path="interval" type="number">
  Interval between recurrences (e.g., every 2 weeks with `freq: "WEEKLY"` and `interval: 2`)
</ParamField>

<ParamField path="count" type="number">
  Number of occurrences before the recurrence ends
</ParamField>

<ParamField path="until" type="Date">
  Date when the recurrence ends
</ParamField>

<ParamField path="wkst" type="string">
  Week start day (e.g., `MO` for Monday)
</ParamField>

<ParamField path="byday" type="string[]">
  Days of the week (e.g., `["MO", "WE", "FR"]`)
</ParamField>

<ParamField path="bymonthday" type="number[]">
  Days of the month (e.g., `[1, 15]` for the 1st and 15th)
</ParamField>

<ParamField path="bymonth" type="number[]">
  Months of the year (e.g., `[1, 6, 12]` for January, June, December)
</ParamField>

    <ParamField path="summary" type="string">
